How a HashMap Works
the happy path — before collisions

A HashMap stores key-value pairs. It uses a hash function to convert any key into an integer, then uses hash(key) % capacity to pick a bucket index. When every key maps to a unique slot — pure O(1).

1
Hash the keyRun "alice" through hash fn → integer 2,847,362
2
Modulo by capacity2,847,362 % 8 = slot 2 → store pair at index 2
3
Retrieve in O(1)hash("alice") % 8 = 2 → jump to index 2 instantly

Load Factor
the number that controls collision frequency
load factor = entries ÷ capacity Java default: 0.75
0.0 empty 0.5 0.75 ◀ 1.0 full
≤ 0.5
Low Collision Risk Rare collisions. O(1) guaranteed. Half the table sits empty — wastes memory.
0.75
Java's Sweet Spot Managed collisions. O(1) amortized. HashMap resizes and rehashes at this point.
≥ 0.9
Collision Cascade ⚡ Long probe chains. Degrades toward O(n). Never let it reach here.
