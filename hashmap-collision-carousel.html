How a
HashMap
Works
before collisions — the happy path

A HashMap stores key-value pairs. To find where a key lives, it runs it through a hash function and uses hash(key) % capacity to get a bucket index. In the best case, every key maps to a unique slot — O(1) get and put.

1
Hash the keyRun "alice" through hash function → integer 2847362
2
Modulo by capacity2847362 % 8 = slot 2 → store pair at index 2
3
Retrieve in O(1)hash("alice") % 8 = 2 → jump directly to index 2

Load
Factor
the number that controls everything
LOAD FACTOR = entries / capacity Java default: 0.75
0.0 empty 0.5 0.75 ◀ 1.0 full
≤ 0.5
Low Collision Risk Collisions rare. O(1) guaranteed. Wastes memory — half the table sits empty.
0.75
Java's Sweet Spot Collisions exist but managed. O(1) amortized. Triggers resize + rehash at this threshold.
≥ 0.9
Collision Cascade ⚡ Long probe chains. O(n) worst case. Degrades to linear scan. Never let it reach here.
