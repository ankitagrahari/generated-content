What is
Big O?
the definition that changes how you code

Big O notation describes how an algorithm's performance scales as input size n grows toward infinity. It ignores constants, ignores hardware — it only asks one question: what happens when the data gets huge?

📏
It measures growth rate, not speed O(n) doesn't mean "slow" — it means operations grow linearly with input.
✂️
Constants are dropped O(2n) and O(500n) both simplify to O(n). Only the shape matters.
💀
Worst-case is default Unless stated otherwise, Big O describes the worst-case scenario.

🟠 DECENT — best possible for sorting
The theoretical floor for comparison-based sorting. You can't sort faster than O(n log n) using comparisons — it's mathematically proven. Slightly worse than linear, but still excellent at real-world scale.
